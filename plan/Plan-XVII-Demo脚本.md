## 版本一：3 分钟 Demo（评委快览版）

时间控制: 每场景约 22 秒，共 8 场景。

### 场景 1: 开场定位（20 秒）

**用户输入:** 无（打开 Dashboard 即可）

**屏幕指向:**
- 鼠标缓慢从左到右划过三栏：
  1. 左侧 Chat 栏（Chief 对话 + 建议 chips）
  2. 中间资产面板（3 张统计卡 + 资产列表 + 资产曲线图）
  3. 右侧 Agent 作战室（6 张 Agent 卡片 + Bitget MCP Skills bar + 调度日志）

**口播:**
> Decision Brain 不是自动下单机器人，而是交易 Agent 的长期决策记忆层。它解决的问题很简单：你买一个资产时有自己的逻辑和估值判断，但市场波动后很容易忘记当初为什么买。我们让 Agent 不只看当下行情，而是能回看仓位、成本、目标、原始 thesis 和计划边界。

---

### 场景 2: 研究 BTC（25 秒）

**用户输入:** `研究 BTC`

**屏幕指向:**
- 点击输入框，输入"研究 BTC"，点击发送
- 等待右侧 Agent 卡片逐个亮起（Macro → Market Intel → News → Sentiment → Technical）
- 指向 Chief 回复中的研究摘要
- 指向右侧调度日志中出现的 Agent fanout 记录

**口播:**
> Bitget MCP Skills 是 Decision Brain 的市场感知层。宏观、市场情报、新闻、情绪和技术面，不是大模型凭空编的，而是通过 MCP 工具进入不同 Agent。Decision Brain 再把这些感知结果变成可追溯的决策上下文。

---

### 场景 3: 记录投资目标与仓位（30 秒）

**用户输入 1:** `我的目标是长期囤 10 个 BTC`

**屏幕指向:**
- Chief 回复确认记录目标
- 中间资产面板出现 BTC 条目

**用户输入 2:** `我现在已经有 3 个 BTC，平均成本 60000。我的逻辑是长期配置 BTC，不做短线。`

**屏幕指向:**
- 中间资产面板 BTC 行：units=3, avg cost=$60,000
- 指向统计卡"组合估值"数值变化
- Chief 回复中 thesis 文本高亮区域

**口播:**
> 这一步是关键区别。Decision Brain 不是只回答"BTC 怎么样"，而是把用户自己的投资目标记下来：目标 10 个 BTC，现在 3 个，成本 $60,000，thesis 是长期配置不做短线。后面每一次加仓、卖出、复盘，都会回到这个上下文。

---

### 场景 4: 市场下跌，用户恐慌（25 秒）

**用户输入:** `现在跌得好厉害，我有点想把 BTC 卖掉。`

**屏幕指向:**
- 输入框输入消息并发送
- 等待 Chief 回复出现
- 逐行指向回复中的关键段落：
  - "原始目标：长期囤 10 BTC"
  - "当前进度：3 / 10"
  - "原始 thesis：长期配置 BTC，不做短线"
  - "先别急着执行"
- 右侧 Agent 作战室本轮调用状态

**口播:**
> 这是 Demo 最重要的一幕。用户不是单纯问行情，而是情绪上想偏离原计划。Decision Brain 先把他拉回最初的投资原则：你原来想囤 10 个，现在只有 3 个；你当时的逻辑是长期配置，不是短线。所以系统不直接说卖或不卖，而是先判断 thesis 有没有失效。这就是投资初心护栏。

---

### 场景 5: 护栏回复展示（25 秒）

**屏幕指向:**
- 停留 5 秒让评委看清 Chief 回复全文
- 逐段高亮：
  - "先别急着执行" + panic flag 识别
  - thesis 回顾段落
  - 计划边界（底仓规则、卖出条件）
  - 克制选项（暂不卖 / 小比例降风险 / 设置复查条件）
  - "数据来源"标注（Bitget MCP trace 引用）

**口播:**
> 系统给出的是克制选项，不是直接喊买卖。它告诉你：thesis 有没有变、估值偏离了多少、计划边界在哪里、什么情况才该卖。而且每一步都有数据来源可追溯。这就是 Decision Brain 和普通行情机器人的区别。

---

### 场景 6: 展示 Trace（20 秒）

**屏幕指向:**
- 右侧 Agent 作战室下方的 trace feed
- 逐条展示本轮调用的 trace 条目：Macro / Market Intel / News / Sentiment / Technical 各自的数据来源
- 高亮 Bitget MCP tool 名称

**口播:**
> 每个 Agent 的结论都可以追溯到具体数据来源。Macro Agent 看什么指标，Sentiment Agent 读什么情绪信号，Technical Agent 分析什么结构。不是黑箱，是可以审计的决策链。

---

### 场景 7: 对话导出（15 秒）

**屏幕指向:**
- 点击右上角"导出对话"按钮
- 下载的 Markdown 文件打开预览
- 展示文件包含：完整对话、Agent fanout、trace、时间戳

**口播:**
> 对话可以一键导出为 Markdown。完整记录用户怎么说的、系统怎么判断的、每个 Agent 做了什么、数据来源是什么。不是只有"买"或"卖"两个字，而是一个可回溯的决策记录。

---

### 场景 8: 收束 + Harness（10 秒）

**屏幕指向:**
- 切到终端，运行 `npm run demo:thesis-guard -- --reset`
- 展示终端输出：8 项检查全部 PASS

**口播:**
> 最后，这个 Demo 不是只靠现场聊碰运气。我们有固定 harness：重置状态、写入目标和仓位、模拟下跌、触发恐慌卖出、检查系统回看 thesis。这个能力既能在 UI 里展示，也能被测试脚本验证。

**用户输入:** 无（打开 http://localhost:4177/ 先看登录页，点击"立即体验"进入 Dashboard）

**屏幕指向:**
- 登录页：3D 线框脑模型旋转 + HUD 四角 + 网格背景
- 点击"立即体验"进入 Dashboard
- 鼠标缓慢从左到右划过三栏，每栏停留 3 秒：
  1. 左侧 Chat（绿色 Chief 头像 + "我是 Chief 决策官"开场语 + 建议 chips）
  2. 中间资产面板（3 张统计卡 + 资产曲线图 + 资产列表）
  3. 右侧 Agent 作战室（6 张 Agent 卡片状态灯 + Bitget MCP Skills 区域 + 调度日志 + trace feed）

**口播:**
> 这是一个交易 Agent 的长期决策记忆层。我们观察到一个真实痛点：用户买代币时通常有当时的判断——为什么买、估值怎么看、持有多久。但市场一回撤，人很容易忘记一开始的投资原则，被短期情绪牵着走。
>
> Decision Brain 把"当初为什么买"和"现在该不该偏离计划"记下来、调出来、解释清楚。左侧是 Chief 对话栏，中间是实时资产看板，右侧是 Agent 作战室。Bitget MCP Skills 提供市场感知，Decision Brain 提供长期记忆和计划复盘。

>
> 这些 Agent 不直接给交易指令，而是把市场变化、情绪变化和技术状态提供给 Chief。Chief 收到各 Agent 的结论后再综合给出回复。注意每个判断都有数据来源，不是大模型编的。

### 场景 3: 记录投资目标与仓位（50 秒）

**用户输入 1:** `我的目标是长期囤 10 个 BTC`

**屏幕指向:**
- 输入并发送

- 指向 Chief 回复中的投资目标存储确认

**用户输入 2:** `我现在已经有 3 个 BTC，平均成本 60000。我的逻辑是长期配置 BTC，不做短线。`

**屏幕指向:**

- 中间资产面板立即更新：
  - 统计卡"组合估值"显示 $183,000（3 * $61,000）
  - 统计卡"持仓"从 0 → 1
  - 资产列表出现 BTC 行：3 BTC / 成本 $60,000 / 现价 $61,000
  - 资产曲线图更新，标题显示"资产曲线 — BTC 总计 $183,000"
- Chief 回复包括：仓位已记录 + thesis 已存档 + 目标进度 3/10
- 指向 Chief 回复中的"原始 thesis：长期配置 BTC，不做短线"

**口播:**
> 这一步展示了 Decision Brain 和普通聊天机器人的关键区别。它不是只回答"BTC 怎么样"，而是把用户自己的投资目标、仓位、成本和 original thesis 记入长期记忆。
>
> 目标是 10 个 BTC，目前 3 个 BTC，成本 $60,000。thesis 是"长期配置，不做短线"。这些信息不是存在对话里就丢了，而是存在仓位记忆系统里。后面用户每一次做买卖决策，Chief 都会先回去读这段记忆。

### 场景 4: 市场下跌，用户恐慌（50 秒）

**用户输入:** `现在跌得好厉害，我有点想把 BTC 卖掉。`

**屏幕指向:**
- 输入框输入消息，发送
- 等待 1-2 秒，注意观察：
  - Chief 状态变为"思考中"
  - 右侧 Agent 作战室本轮 fanout：Memory Agent + Valuation Agent 亮起
  - 调度日志新增条目
- Chief 回复出现后，逐段指向：
  1. panic flag 识别："先别急着执行"
  2. 原始目标回顾："你设定的目标是长期囤 10 BTC，当前进度 3/10"
  3. thesis 回顾："你的原始 thesis 是长期配置 BTC，不做短线"
  4. 计划边界："底仓规则：最少保留 2 BTC。卖出条件：thesis 失效时复盘 / 估值止盈区 / 不得因单日下跌清仓"
  5. 当前判断："当前更多是价格波动触发的情绪反应，而非 thesis 失效"
  6. 克制选项："暂不卖 / 小比例降低风险 / 设置复查条件"

**口播:**
> 这是整个 Demo 最关键的一幕。用户不是来问行情，而是情绪上想偏离原计划。
>
> Decision Brain 怎么处理？第一步，识别情绪——系统判定这是 panic sell，不是理性复盘。第二步，回看记忆——你原来要囤 10 个，现在才 3 个；你当时说长期配置不做短线。第三步，检查计划边界——底仓规则是最少留 2 个，卖出条件是 thesis 失效或到止盈区，不是单日下跌。
>
> 最后，系统不是替用户做决定，而是给出三个克制选项。这就是投资初心护栏：不是拦着你不卖，而是让你在卖之前回看当初为什么买。

---

### 场景 5: 护栏回复详读（30 秒）

**屏幕指向:**
- 停留，让评委仔细看 Chief 回复全文
- 重点高亮：
  - 回复开头 panic 识别段落
  - thesis 回顾 + 目标进度段落
  - 计划边界段落（底仓规则 yellow tag）
  - 选项列表段落
  - 底部"数据来源"标注（含 Bitget MCP tool 引用）

**口播:**
> 注意这几层信息：panic 识别是第一层——先判断你是理性还是情绪化。thesis 回顾是第二层——你的原始逻辑还记得吗。计划边界是第三层——你的底仓规则和卖出条件是什么。克制选项是第四层——不替你做决定，但给出判断框架。
>
> 这四层加起来，就是 Decision Brain 的核心：不是行情分析，而是决策护栏。

**口播:**
> 每一步判断都有 trace 可回溯。Memory Agent 召回仓位上下文，Macro Agent 看宏观风险偏好，Sentiment Agent 读恐惧贪婪指数，Valuation Agent 算估值偏离，Technical Agent 看价格结构。
>
> 这些 Agent 背后是 Bitget MCP Skills 提供的数据。不是黑箱预测，而是可审计的决策链。

## 演示后备方案

如果某个环节出错：

| 问题 | 备选方案 |
|------|---------|
| 服务器连不上 | 提前准备截图/录屏备选 |
| Agent fanout 没响应 | 使用 mock 模式：`?mock=1` |
| harness 某项 FAIL | 提前跑一次确保通过；若现场 FAIL，展示上次通过截图 |
| UI 布局错乱 | 提前用 Chrome DevTools 确认窗口宽度 >= 1400px |
| Bitget MCP 没连接 | 诚实说明"MCP 在当前环境未配置"，展示 Skills bar 区域作为架构示意 |
