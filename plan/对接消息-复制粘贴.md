## 📋 复制段 4 —— 仅当有第三人，发给 C 组（存储+部署）

你负责 C 组：把存储和部署从 A 组拆出来，让 A 专心做 Agent。打开文档看：

- §4 存储改造（核心任务）：data-store.mjs 现在是「整个 state 当一个 JSON blob 读写」，你抽一层 storage backend 接口（readBlob/writeBlob），做两个实现——本地用文件，Vercel 用 KV (Upstash Redis)。12 个 service 一行都不用改。
- §8 Vercel 部署：vercel.json、配 LLM 和 KV 的环境变量、验证 /api/health。
- §6 组 A 里的 A5（存储双实现）和 A6（Vercel 部署）就是你的活。

你的交付标准：
1. 不设 KV 环境变量时走文件存储；设了就走 KV——同一套代码，靠环境探测切换
2. evaluate 一个资产后重启进程，再查 /api/state 数据还在（持久化成功），文件和 KV 两种都要验
3. Vercel URL 可公开访问，/api/health 返回 ok
4. KV 当天搞不定就回退 ngrok 本地（零改动）保证有公开链接，别卡死

跟 A 组确认好 DataStore 的 load/save 怎么调你的 backend，这是你俩唯一的接口。
